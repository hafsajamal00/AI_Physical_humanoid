# Quickstart: Adding Module 3 to Docusaurus

**Feature**: 003-isaac-ai-brain
**Purpose**: Step-by-step guide to integrate Module 3 content into existing book_frontend Docusaurus site

## Prerequisites

- Docusaurus 3.x already set up in `book_frontend/` (from Modules 1-2)
- @docusaurus/theme-mermaid plugin installed (from Module 2)
- Modules 1 and 2 content already deployed
- Node.js and npm installed

## Step 1: Create Module 3 Directory Structure

```bash
cd book_frontend

# Create Module 3 docs directory
mkdir -p docs/module-3

# Create Module 3 static assets directory (for future images if needed)
mkdir -p static/img/module-3
```

**Expected Result**: New directories exist at `book_frontend/docs/module-3/` and `book_frontend/static/img/module-3/`

---

## Step 2: Create Module 3 Category Configuration

Create `book_frontend/docs/module-3/_category_.json`:

```json
{
  "label": "Module 3: The AI-Robot Brain (NVIDIA Isaac™)",
  "position": 3,
  "link": {
    "type": "generated-index",
    "description": "Master advanced perception and navigation using NVIDIA Isaac Sim, Isaac ROS, and Nav2 for humanoid robots."
  }
}
```

**Expected Result**: Module 3 will appear in sidebar after Modules 1-2 with auto-generated index page

## Content Authoring Workflow

Once placeholder files are set up, follow this workflow to author content:

1. **Read Contract**: Review relevant chapter outline in `specs/003-isaac-ai-brain/contracts/chapter-N-outline.md`
2. **Draft Content**: Replace "Coming Soon" section with full content following contract structure
3. **Add Diagrams**: Insert Mermaid diagrams and markdown tables per contract requirements
4. **Add References**: Include external links to NVIDIA Isaac, ROS 2 Nav2 documentation
5. **Validate Build**: Run `npm run build` to ensure Mermaid renders correctly
6. **Test Navigation**: Run `npm start` and verify all links, diagrams, tables display properly
7. **Cross-Reference**: Ensure bridge paragraphs connect chapters (Ch1 → Ch2, Ch2 → Ch3)

---

## Common Issues & Solutions

**Issue**: Module 3 doesn't appear in sidebar
- **Solution**: Check `_category_.json` has valid JSON syntax, `position: 3` is unique

**Issue**: Mermaid diagram doesn't render
- **Solution**: Verify Mermaid syntax with [Mermaid Live Editor](https://mermaid.live/), ensure code block uses \`\`\`mermaid

**Issue**: Chapter shows 404 error
- **Solution**: Verify frontmatter has `sidebar_position` set, filename matches slug (chapter-1-isaac-sim.md)

**Issue**: Build fails with "Unexpected character" error in table
- **Solution**: Escape `<` and `>` characters in markdown tables (use "less than" or "greater than" text instead)
